# Cheatsheets

A growing collection of clean, privacy-first developer cheatsheets — read them on
the web, or download each as **PDF** or **Markdown**.

**Live site:** https://michalaferber.github.io/cheatsheets/

Built with **Jekyll** and the [mkdocs-jekyll](https://github.com/vsoch/mkdocs-jekyll)
theme (light, mobile-first, sidebar nav + search), deployed to GitHub Pages.

## Repository layout

```
.
├── _config.yml               # Jekyll config (remote theme, collections, colours)
├── Gemfile / Gemfile.lock     # Ruby dependencies (github-pages gem)
├── index.md                  # Home page (generated by scripts/gen-nav.mjs)
├── _data/
│   └── toc.yml               # Sidebar navigation (generated)
├── _docs/                    # One Markdown page per cheatsheet (the source)
│   ├── linux-commands.md
│   └── …
├── assets/downloads/         # PDF + Markdown download files, per cheatsheet
├── scripts/
│   └── gen-nav.mjs           # Rebuilds _data/toc.yml + index.md from _docs/
├── .github/workflows/
│   └── deploy-pages.yml      # Builds Jekyll and deploys to Pages
├── LICENSE
└── README.md
```

## Adding a cheatsheet

1. Create `_docs/<slug>.md` with front matter, then the content:

   ```markdown
   ---
   title: "Docker"
   description: "Build, run, and clean up containers."
   category: "DevOps"     # Shell & CLI | Editors | DevOps | Languages & Data | macOS
   ---
   # Docker

   [Download PDF]({{ '/assets/downloads/docker.pdf' | relative_url }}){: .pdf-link }

   ## Images

   | Command | Action |
   | --- | --- |
   | `docker ps` | list running containers |

   > **Tip:** blockquotes render as callout boxes.
   ```

2. (Optional) Drop a `assets/downloads/<slug>.pdf` for the download link.
3. Regenerate the nav + home page:

   ```bash
   node scripts/gen-nav.mjs
   ```

4. Commit and push to `main` — the workflow rebuilds and redeploys.

## Local development

```bash
bundle install
bundle exec jekyll serve      # http://localhost:4000/cheatsheets/
```

## Deployment

Handled by [`.github/workflows/deploy-pages.yml`](.github/workflows/deploy-pages.yml):
it installs the gems, runs `jekyll build` (with the correct `--baseurl`), and
deploys `_site/`. Repo setting: **Settings → Pages → Source → GitHub Actions**.

## Privacy & analytics

The site is privacy-first: no cookies, no ad trackers, and no personal data leaves
your browser. Aggregate traffic counts come from a self-hosted
[Plausible](https://plausible.io) instance — it is cookie-less, stores no personally
identifiable information, and does no cross-site tracking. There is nothing to consent
to and nothing to opt out of.

## Deviations from the TGWAB Dev Standards

This is a **Class C micro-project** — a static content site on GitHub Pages — so a few
of the standard's requirements are scoped down or waived. For transparency:

- **No footer credit line.** The standard's "Created by…" footer is intentionally
  omitted here.
- **The GitHub octocat is kept as the brand mark** rather than a bespoke logo.
- **No Privacy Policy or Terms pages.** The site is static and collects no personal
  data (see [Privacy & analytics](#privacy--analytics)), so there is nothing for those
  pages to govern.
- **§11 (SEO/plumbing kit) and §12 (`_headers` / CSP) do not apply.** They target
  Cloudflare-hosted apps; GitHub Pages cannot serve custom response headers, and a
  micro-project does not need the full SEO kit.
- **Analytics is self-hosted Plausible**, following the standard's
  privacy-friendly-analytics guidance.
